# {Product/Ecosystem} — Channel Registry

> Voice register is not governed by style guides. It's governed by structural constraints.
> The channel determines the register. The named author creates warmth. The vocabulary bank bounds what's available.

## The Two Axes

**Axis 1: Named-author warmth gradient**
The closer to a named human, the warmer. The closer to a system process, the colder.

**Axis 2: Vocabulary ceiling**
Each channel has an implicit maximum register -- the highest lore/personality permitted by its function.

## The Tonal Gap Rule

Communities read register shifts between hype and accountability as a values signal.
If feature announcements are warm and bug acknowledgments are cold, the community interprets: "they care more about selling than fixing."

**The voice should NOT drop for problems.**

## The Load-Bearing Test

Before using a world term in operational comms:

1. **Is this the only name for the thing?** -> Use it. It's the identifier.
2. **Does the community use this term in conversation?** -> Use it. It's native speech.
3. **Would removing it make the message clearer?** If yes -> it's decoration. Cut it.
4. **Is it replacing information or carrying it?** Carrying -> keep. Replacing -> cut.
