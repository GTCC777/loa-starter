# Communication Context (Base)

Reference material for crafting announcements in crypto community contexts.

---

## High-Signal Announcement Patterns

<!-- @table:high-signal-patterns [merge-mode:append] -->

Patterns that build credibility over time:

| Pattern | Effect |
|---------|--------|
| Dates from git history | Verifiable, builds trust |
| "Never shipped" honesty | Prevents narrative gaps |
| Named external reasons | Shows awareness, not excuses |
| Practical action items with deadlines | Treats audience as adults |
| One-line feature descriptions from code | Demonstrates understanding |
| Past tense for removed features | Clear about what's gone |

<!-- @table:high-signal-patterns:end -->

---

## Low-Signal Patterns (Anti-Credibility)

<!-- @table:low-signal-patterns [merge-mode:append] -->

Patterns that erode trust, especially in crypto communities where over-promising is endemic:

| Pattern | Problem | Community reads it as |
|---------|---------|----------------------|
| "Stay tuned" | Forward-looking promise | "They're about to announce something" |
| "Exciting things ahead" | Hype without substance | "They have nothing to show" |
| "Unfortunately" | Apologetic framing | "They know they messed up" |
| "We've been cooking" | Unverifiable claim | "Vapor" |
| "Trust the process" | Asks for faith over evidence | "They can't deliver" |
| "This is just the beginning" | Shifts goalposts | "The current thing isn't enough" |
| Internal screenshots | Premature reveal | "When is this shipping?" (forever) |

<!-- @table:low-signal-patterns:end -->

## Announcement Type Patterns

### Removal / Sunset

**Credibility order:** Action → History → Reason → Deadline → Survivors → Close

People process removals better when they understand:
1. What's happening (the action)
2. What it was (grounded in what actually existed)
3. Why now (external, concrete)
4. What to do (practical)

### Release / New Feature

**Credibility order:** What shipped → What it does → How to access → Limitations

New features announce themselves through usage. The announcement's job is accurate description, not salesmanship.

### Structural Change

**Credibility order:** What moved → User impact → What stays the same

People fear change means loss. Leading with what stays the same reduces anxiety.

---

## Community Memory

Crypto communities have long memories and screenshot culture. Assume every announcement will be:
- Screenshot'd and shared out of context
- Compared to previous announcements for contradictions
- Referenced 6+ months later as evidence of promises
- Used in DMs to argue for refunds or special treatment

**Implication:** Every sentence must be defensible in isolation, months from now, by someone who wants to prove you lied.
